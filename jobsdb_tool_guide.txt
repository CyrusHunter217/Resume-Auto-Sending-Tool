# JobsDB 自动化求职工具开发指南

## 目标
使用 Python 和 DeepSeek API 开发一个工具，实现以下功能：
1. 分析 JobsDB 岗位要求。
2. 针对性修改简历。
3. 自动/半自动投递。

## 整体策略
分为三个阶段：
1. **阶段一：环境准备与项目初始化**
   - 搭建 Python 开发环境。
   - 创建基础 GUI 窗口。
2. **阶段二：核心功能开发**
   - 实现 `分析岗位描述` 和 `生成定制简历` 功能。
3. **阶段三：自动化投递与整合**
   - 实现自动填写 JobsDB 申请表功能。
   - 整合所有模块到 GUI 中。

## 阶段一：环境准备与项目初始化

### 目标
创建一个带有输入框和按钮的空白窗口。

### 前置准备
1. 安装 Python（从 [python.org](https://www.python.org/) 下载）。
2. 打开 Cursor 编辑器。

### 步骤 1：创建 GUI 窗口
1. 创建新文件 `job_app_bot.py`。
2. 使用以下提示词生成代码：
   ```
   提示词 1：
   我是一个编程新手，想用 Python 做一个求职小工具。我的目标是：
   1. 有一个简单的窗口（GUI）。
   2. 窗口里有一个大文本框，让我可以粘贴 JobsDB 的岗位描述。
   3. 有一个“生成定制简历”的按钮。
   4. 点击按钮后，能在另一个文本框里看到结果（目前先显示“正在处理...”）。

   请使用 `tkinter` 库来实现这个基础窗口，因为它是 Python 自带的，最简单。请给我完整的代码，并附上详细的注释，告诉我每一部分是干什么的。
   ```
3. 将生成的代码复制到 `job_app_bot.py` 中。
4. 运行文件：`python job_app_bot.py`。

## 阶段二：核心功能开发（AI 分析与简历生成）

### 目标
接入 DeepSeek API，实现“输入岗位描述 → 输出定制简历”的核心逻辑。

### 前置准备
1. 获取 DeepSeek API Key。
2. 安装 `requests` 库：`pip install requests`。

### 步骤 2：编写 AI 调用函数
1. 使用以下提示词生成代码：
   ```
   提示词 2：
   我想在上一个 GUI 程序的基础上，添加一个函数，用它来调用 DeepSeek API。
   这个函数的功能是：
   1. 接收两个输入：一个是“岗位描述”文本，另一个是“我的原始简历”文本。
   2. 使用 `requests` 库向 DeepSeek API 发送一个 POST 请求。
   3. Prompt 应该这样设计：“你是一位专业的求职顾问。请根据下面的【岗位描述】，重写我的【原始简历】，突出与岗位最匹配的技能和经验。使用专业、简洁的语言，不要超过一页。”
   4. 函数最终返回 API 的响应文本。

   请帮我：
   a) 在代码顶部添加必要的 import 语句 (import requests, json)。
   b) 定义一个名为 `generate_custom_resume` 的函数，实现上述功能。
   c) 函数中需要有一个地方让我填入我的 API Key (用一个变量代替)。
   d) 处理可能的网络错误，并给出提示。
   ```
2. 将生成的代码复制到 `job_app_bot.py` 中。
3. 填入你的 DeepSeek API Key。

### 步骤 3：整合函数到 GUI
1. 使用以下提示词生成代码：
   ```
   提示词 3：
   现在，我需要你帮我修改 GUI 部分的代码。
   1. 在窗口中，增加一个“我的原始简历”的文本框（可以是一个大的输入框）。
   2. 修改“生成定制简历”按钮的 `command`，让它调用我们刚刚写的 `generate_custom_resume` 函数。
   3. 调用时，需要把“岗位描述”文本框和“我的原始简历”文本框中的内容作为参数传进去。
   4. 因为 API 调用需要一点时间，为了防止窗口假死，请使用 `threading` 库来在新线程中执行这个任务。
   5. `generate_custom_resume` 函数返回的结果，要显示在最开始那个“结果”文本框里。

   请给我完整的、更新后的 GUI 代码块。
   ```
2. 将生成的代码替换到 `job_app_bot.py` 中。
3. 运行文件测试功能。

## 阶段三：自动化投递与整合

### 目标
实现自动填写 JobsDB 在线申请表的功能。

### 前置准备
1. 安装 `selenium` 库：`pip install selenium`。
2. 下载并配置对应浏览器的 `webdriver`（如 ChromeDriver）。

### 步骤 4：编写自动投递函数
1. 使用以下提示词生成代码：
   ```
   提示词 4：
   我想再增加一个自动投递的功能。我需要一个函数，能用 `selenium` 库自动打开 Chrome 浏览器，访问一个指定的 JobsDB 申请页面，并填写表单。
   这个函数 `auto_apply` 需要接收以下参数：
   - job_url: 岗位申请页面的 URL。
   - custom_resume_text: 我们生成的定制简历文本。
   - my_info: 一个包含我个人信息的字典，比如 {'name': '张三', 'email': 'xxx@xxx.com', 'phone': '12345678'}。

   函数的大致流程：
   1. 初始化一个 Chrome 浏览器实例。
   2. 访问 job_url。
   3. 找到“姓名”输入框，输入 my_info['name']。
   4. 找到“邮箱”输入框，输入 my_info['email']。
   5. 找到“电话”输入框，输入 my_info['phone']。
   6. 找到“工作经验”或“自我介绍”的文本域，输入 custom_resume_text。
   7. （可选）找到“上传简历”按钮，上传一个文件（这个比较复杂，可以先跳过）。
   8. 最后，找到“提交”按钮并点击。

   注意：JobsDB 的网页元素 ID 可能会变化，你可以用 `find_element_by_name` 或 `find_element_by_xpath` 来定位。如果定位不到，可以打印出页面源码或给出错误提示。

   请帮我编写这个 `auto_apply` 函数，并告诉我如何准备 `webdriver`。
   ```
2. 将生成的代码复制到 `job_app_bot.py` 中。
3. 配置 `webdriver` 路径。

### 步骤 5：最终整合到 GUI
1. 使用以下提示词生成代码：
   ```
   提示词 5：
   现在，请帮我最后一次更新 GUI 代码：
   1. 在窗口底部增加一个“岗位申请 URL”的输入框。
   2. 增加一个“自动投递”按钮。
   3. 这个按钮的 `command` 要调用 `auto_apply` 函数。
   4. `auto_apply` 的参数：
      - job_url: 从“岗位申请 URL”输入框获取。
      - custom_resume_text: 从“结果”文本框获取。
      - my_info: 你可以在代码里直接定义一个字典，包含我的姓名、邮箱、电话等固定信息。
   5. 同样，为了防止窗口假死，请用 `threading` 来调用 `auto_apply`。

   请给我最终的、完整的 GUI 代码块。
   ```
2. 将生成的代码替换到 `job_app_bot.py` 中。
3. 填入你的个人信息到 `my_info` 字典中。
4. 运行文件测试完整功能。

## 总结与后续

### 工具工作流程
1. 从 JobsDB 复制岗位 URL 和岗位描述。
2. 将岗位描述粘贴到工具的“岗位描述”框。
3. 确保“我的原始简历”框里有你的基础简历。
4. 点击“生成定制简历”，等待 AI 生成结果。
5. 将岗位的申请 URL 粘贴到“岗位申请 URL”框。
6. 点击“自动投递”，然后看着 Chrome 自动为你填写并提交申请表单。

### 重要提示
- 网站结构经常变化，`selenium` 的定位器可能会失效，需要根据实际情况调整。
- 频繁的自动化操作可能会被网站识别为机器人，导致账号被限制。建议加入适当的延时，并谨慎操作。
